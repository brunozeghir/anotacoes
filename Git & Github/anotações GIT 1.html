Ideal é usar o mesmo e-mail e usuário do github no git. Caso tenha feito diferente, tem como apagar o e-mail e usuário e cadastrar novamente. Os comandos abaixo irão apagar o email e usuário. git config --global --unset user.email (Apaga o email). git config --global --unset user.name (Apaga o usuário). git config --list (irá mostrar a lista de config do git). ex: usuário e email. Após criarmos o repositório no github, copiamos o link fornecido. Vamos ao git, digitaremos o seguinte comando. git remote add origin https://github.... origin é um nome que a gente dá para o link que iremos enviar nossos arquivos. Após, colocaremos o seguinte código no git git push origin master confirma a ação caso apareça algo para confirmar. Pronto. Caso ao enviar um arquivo que já tenha no github mas vc editou, e dê erro por existir modificações diferentes na mesma linha, vc deverá trazer o arquivo do github para sua máquina para fazer a alteração do conflito e depois enviar novamente. Para trazer o arquivo em conflito, usaremos o comando seguinte: git pull origin master Assim terá baixado o arquivo a ser modificado.
